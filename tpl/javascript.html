概览

引入文件

JavaScript 插件引入（使用 lxui.js 或压缩版的 lxui.min.js）即可。还提供独立的插件可供选择自行添加。

建议使用压缩版的 JavaScript 文件

lxui.js 和 lxui.min.js 都包含了所有插件，你在使用时，只需选择一个引入页面就可以了。

组件的 data 属性

不要在同一个元素上同时使用多个插件的 data 属性。例如，button 组件不能同时支持工具提示和控制弹窗。如果需要的话，可以在其外面包裹一个额外的元素。

插件之间的依赖关系

某些插件和 CSS 组件依赖于其它插件。如果你是单个引入每个插件的，请确保在文档中检查插件之间的依赖关系。注意，所有插件都依赖 jQuery （也就是说，jQuery必须在所有插件之前引入页面）。

data 属性

你可以仅仅通过 data 属性 API 就能使用所有的 LXUI 插件，无需写一行 JavaScript 代码。这是 LXUI 中的推荐 API，也应该是你的首选方式。

在某些情况下可能需要将此功能关闭。因此，我们还提供了关闭 data 属性 API 的方法，即解除以 data-api 为命名空间并绑定在文档上的事件。就像下面这样：

$(document).off('.data-api')

另外，如果是针对某个特定的插件，只需在 data-api 前面添加那个插件的名称作为命名空间，如下：

$(document).off('.alert.data-api')

编写方式的 API

所有公开的 API 都是支持单独或链式调用方式，并且返回其所操作的元素集合（注：和jQuery的调用形式一致）。

$('.btn.danger').button('toggle').addClass('fat')

所有方法都可以接受一个可选的 option 对象作为参数，或者一个代表特定方法的字符串，或者什么也不提供（在这种情况下，插件将会以默认值初始化）。

$('#myModal').modal()                      // 以默认值初始化
$('#myModal').modal({ keyboard: false })   // initialized with no keyboard
$('#myModal').modal('show')                // 初始化后立即调用 show 方法

避免命名冲突

某些时候可能需要将 LXUI 插件与其他 UI 框架共同使用。在这种情况下，命名空间冲突随时可能发生。如果不幸发生了这种情况，你可以通过调用插件的 .noConflict 方法恢复其原始值。（如果打算使用 bootstrap 框架就不需要加载 lxui.js/lxui.min.js）

var LxuiButton = $.fn.button.noConflict() // return $.fn.button to previously assigned value
$.fn.bootstrapBtn = LxuiButton            // give $().LxuiBtn the LXUI functionality

事件

为大部分插件所具有的动作提供了自定义事件。一般来说，这些事件都有不定式和过去式两种动词的命名形式，例如，不定式形式的动词（例如 show）表示其在事件开始时被触发；而过去式动词（例如 shown ）表示在动作执行完毕之后被触发。

所有以不定式形式的动词命名的事件都提供了 preventDefault 功能。这就赋予你在动作开始执行前将其停止的能力。

$('#myModal').on('show.bs.modal', function (e) {
  if (!data) return e.preventDefault() // 阻止弹窗的显示
})

弹出层

弹出层以弹出对话框的形式出现，具有最小和最实用的功能集。

不支持弹出层重叠

千万不要在一个弹出层上重叠另一个弹出层。要想同时支持多个模态框，需要自己写额外的代码来实现。

模态框的 HTML 代码放置的位置

务必将模态框的 HTML 代码放在文档的最高层级内（也就是说，尽量作为 body 标签的直接子元素），以避免其他组件影响模态框的展现和/或功能。

用法

通过 data 属性或 JavaScript 调用弹出层插件，可以根据需要动态展示隐藏的内容。弹出层弹出时还会为 <body> 元素添加 .modal-open ，从而覆盖页面默认的滚动行为，并且还会自动生成一个 .modal-backdrop 元素用于提供一个可点击的区域，点击此区域就即可关闭弹出层。

通过 data 属性使用

不需写 JavaScript 代码也可激活弹出层。通过在一个起控制器作用的元素（例如：按钮）上添加 data-toggle="modal" 属性，或者 data-target="#foo" 属性，再或者 href="#foo" 属性，用于指向被控制的弹出层。

<button type="button" data-toggle="modal" data-target="#myModal">Launch modal</button>

通过 JavaScript 使用

只需一行 JavaScript 代码，即可通过元素的 id myModal 调用弹出层：

$('#myModal').modal(options)

参数设置 (options)

可以将选项通过 data 属性或 JavaScript 代码传递。对于 data 属性，需要将参数名称放到 data- 之后，例如 data-backdrop=""。

名称 类型 默认值 描述
backdrop boolean 或 字符串 'static' true Includes a modal-backdrop element. Alternatively, specify static for a backdrop which doesn't close the modal on click.
keyboard boolean true 键盘上的 esc 键被按下时关闭弹出层。
show boolean true 弹出层初始化之后就立即显示出来。
remote path false

如果提供的是 URL，将利用 jQuery 的 load 方法从此 URL 地址加载要展示的内容（只加载一次）并插入 .modal-content 内。如果使用的是 data 属性 API，还可以利用 href 属性指定内容来源地址。下面是一个实例：

Copy
<a data-toggle="modal" href="remote.html" data-target="#modal">Click me</a>

方法

.modal(options)

将页面中的某块内容作为弹出层激活。接受可选参数 object。

$('#myModal').modal({
  backdrop: true,  //参数默认值 true
  keyboard: false, //参数默认值 true
  show: false      //参数默认值 true
})
.modal('toggle')

手动打开或关闭弹出层。在弹出层显示或隐藏之前返回到主调函数中（也就是，在触发 shown.bs.modal 或 hidden.bs.modal 事件之前）。

$('#myModal').modal('toggle')
.modal('show')

手动打开弹出层。在弹出层显示之前返回到主调函数中 （也就是，在触发 shown.bs.modal 事件之前）。

$('#myModal').modal('show')
.modal('hide')

手动隐藏弹出层。在弹出层隐藏之前返回到主调函数中 （也就是，在触发 hidden.bs.modal 事件之前）。

$('#myModal').modal('hide')

事件

弹出层提供了一些事件用于监听并执行你自己的代码

事件类型 描述
show.bs.modal show 方法调用之后立即触发该事件。如果是通过点击某个作为触发器的元素，则此元素可以通过事件的 relatedTarget 属性进行访问。
shown.bs.modal 此事件在弹出层已经显示出来（并且同时在 CSS 过渡效果完成）之后被触发。如果是通过点击某个作为触发器的元素，则此元素可以通过事件的 relatedTarget 属性进行访问。
hide.bs.modal hide 方法调用之后立即触发该事件。
hidden.bs.modal 此事件在弹出层被隐藏（并且同时在 CSS 过渡效果完成）之后被触发。
loaded.bs.modal 从远端的数据源加载完数据之后触发该事件。 
$('#myModal').on('hidden.bs.modal', function (e) {
  // do something...
})

用法

用 data 属性或者 JavaScript 来实现，原理就是当点击下拉菜单后即给 .dropdown-menu 后加 .open 来显示下拉层内容。

通过 data 属性使用

给链接或按钮加入 data-toggle="dropdown" 来激活下拉菜单。

<div class="dropdown">
  <a data-toggle="dropdown" href="#">Dropdown trigger</a>
  <ul class="dropdown-menu" role="menu" aria-labelledby="dLabel">
    ...
  </ul>
</div>
也可以给下拉菜单按钮添加链接属性，要用 data-target 属性而不是 href="#" 
<div class="dropdown">
  <a id="dLabel" role="button" data-toggle="dropdown" data-target="#" href="/page.html">
    Dropdown <span class="caret"></span>
  </a>

  <ul class="dropdown-menu" role="menu" aria-labelledby="dLabel">
    ...
  </ul>
</div>

通过 JavaScript 使用

用 JavaScript 定义下拉菜单：

$('.dropdown-toggle').dropdown()

data-toggle="dropdown" 属性仍然需要

不管是否用 JavaScript 还是 data 属性来激活下拉菜单，你都需要给按钮或链接 data-toggle="dropdown" 属性，这样才能正确使用触发下拉菜单。

方法

$().dropdown('toggle')

通过对某个特定的导航条或标签页导航切换下拉菜单的打开关闭。

事件

Event Type Description
show.bs.dropdown show 方法调用之后立即触发该事件。如果是通过点击某个作为触发器的元素，则此元素可以通过事件的 relatedTarget 属性进行访问。
shown.bs.dropdown 此事件在下拉菜单已经显示出来之后被触发。如果是通过点击某个作为触发器的元素，则此元素可以通过事件的 relatedTarget 属性进行访问。
hide.bs.dropdown hide 方法调用之后立即触发该事件。
hidden.bs.dropdown 此事件在下拉菜单被隐藏之后被触发。 
$('#myDropdown').on('show.bs.dropdown', function () {
  // do something…
})

用法

需要给添加滚动监听的功能的元素 添加 positon: relative;属性，更多情况下是给 <body> 添加。

通过 data 属性使用

只需将 data-spy="scroll" 添加到被监听的页面元素上（大部分情况是添加到body上），然后将 data-target=".navbar" 添加到导航部分，顶部导航条就拥有了监听滚动的功能。

body {
  position: relative;
}

<body data-spy="scroll" data-target=".navbar-example">
  ...
  <div class="navbar-example">
    <ul class="nav nav-tabs" role="tablist">
      ...
    </ul>
  </div>
  ...
</body>

通过 JavaScript 使用

在你的 CSS 中加入 position: relative; 后，通过JavaScript 启动滚动监听：

$('body').scrollspy({ target: '.navbar-example' })

指定元素ID的要求

必须为导航条中的链接指定相应的目标id。例如，<a href="#home">home</a> 必须和dom中类似 <div id="home"></div> 的页面元素相呼应。

方法

.scrollspy('refresh')

当滚动监听所作用的DOM有增删页面元素的操作时，需要调用下面的refresh方法：

$('[data-spy="scroll"]').each(function () {
  var $spy = $(this).scrollspy('refresh')
})

参数设置

所有参数都可以通过 data 属性或 JavaScript 设置。对于 data 属性，将参数名附着到 data- 后面，就像 data-offset="" 一样。

Name type default description
offset number 10 从顶部到当前滚动位置的像素单位。

事件

Event Type Description
activate.bs.scrollspy 当有新条目被激活，此事件就出发。 
$('#myScrollspy').on('activate.bs.scrollspy', function () {
  // do something…
})

用法

通过JavaScript启动标签页（每个标签都需要被单独激活）：

$('#myTab a').click(function (e) {
  e.preventDefault()
  $(this).tab('show')
})

你还可以通过多种方式激活标签页

$('#myTab a[href="#profile"]').tab('show') // 选择元素的 name
$('#myTab a:first').tab('show') // 选择第一个标签选项
$('#myTab a:last').tab('show') // 选择最后一个标签选项
$('#myTab li:eq(2) a').tab('show') // 选择第3个标签选项（0 是第一个）

标记

无需写任何 JavaScript 代码，只需简单的为页面元素指定 data-toggle="tab" 或 data-toggle="pill" 属性即可激活标签页或其他样式标签页。为 ul 添加 nav 和 nav-tabs classe，即可为其添加标签页样式。

<!-- Nav tabs -->
<ul class="nav nav-tabs" role="tablist">
  <li class="active"><a href="#home" role="tab" data-toggle="tab">Home</a></li>
  <li><a href="#profile" role="tab" data-toggle="tab">Profile</a></li>
  <li><a href="#messages" role="tab" data-toggle="tab">Messages</a></li>
  <li><a href="#settings" role="tab" data-toggle="tab">Settings</a></li>
</ul>

<!-- Tab panes -->
<div class="tab-content">
  <div class="tab-pane active" id="home">...</div>
  <div class="tab-pane" id="profile">...</div>
  <div class="tab-pane" id="messages">...</div>
  <div class="tab-pane" id="settings">...</div>
</div>

过渡事件

在 .tab-pane 后面加上 .fade 使切换标签页选项有渐变过渡效果，第一个标签选项必须有 .in 才能正确显示第一个选项内容。

<div class="tab-content">
  <div class="tab-pane fade in active" id="home">...</div>
  <div class="tab-pane fade" id="profile">...</div>
  <div class="tab-pane fade" id="messages">...</div>
  <div class="tab-pane fade" id="settings">...</div>
</div>

选择加入的功能

出于性能方面的考虑，默认关闭此插件，你需要自己初始化来使用工具提示功能。

在按钮组中加入工具提示需要特殊的设置

当工具提示用在 .btn-group 或者 .input-group内，你必须指定到需要出发的按钮上，而不是父层。

不要试着在隐藏的对象中显示工具提示

当元素对象是 display: none; 调用 $(...).tooltip('show') 将导致工具提示无法正确定位

用法

工具提示插件启动后会生成title层，当被触发元素后提示。

通过JavaScript触发工具提示：

$('#example').tooltip(options)

标记

<!-- HTML to write -->
<a href="#" data-toggle="tooltip" title="Some tooltip text!">Hover over me</a>

<!-- Generated markup by the plugin -->
<div class="tooltip top" role="tooltip">
  <div class="tooltip-arrow"></div>
  <div class="tooltip-inner">
    Some tooltip text!
  </div>
</div>

参数设置

可以通过 data 属性或 JavaScript 传递参数。对于 data 属性，将参数名附着到 data- 后面即可，例如, data-animation=""。

Name Type Default Description
animation boolean true 应用CSS淡入淡出过渡效果到工具提示。
container string | false false

Appends the tooltip to a specific element. Example: container: 'body'. This option is particularly useful in that it allows you to position the tooltip in the flow of the document near the triggering element - which will prevent the tooltip from floating away from the triggering element during a window resize.
delay number | object 0

延迟显示和隐藏工具提示（毫秒） - 不适用于手动触发类型。

placement string | function 'top' 如何定位工具提示 - top | bottom | left | right | auto 。

用法

通过 data 属性

Just add data-toggle="collapse" and a data-target to element to automatically assign control of a collapsible element. The data-target attribute accepts a CSS selector to apply the collapse to. Be sure to add the class collapse to the collapsible element. If you'd like it to default open, add the additional class in.

To add accordion-like group management to a collapsible control, add the data attribute data-parent="#selector". Refer to the demo to see this in action.

通过 JavaScript

手动调用：

$('.collapse').collapse()

参数设置

参数传递可通过 data 属性或 JavaScript。对于 data 属性，需要附加选项名称 data-，例如 data-parent=""。

Name type default description
parent selector false If selector then all collapsible elements under the specified parent will be closed when this collapsible item is shown. (similar to traditional accordion behavior - this dependent on the panel class)
toggle boolean true Toggles the collapsible element on invocation

方法

.collapse(options)

启用折叠功能，可以添加多个参数设置。

$('#myCollapsible').collapse({
  toggle: false
})
.collapse('toggle')

切换一个可折叠的元素来显示或隐藏。

.collapse('show')

显示一个折叠对象。

.collapse('hide')

隐藏一个折叠对象。

事件

Event Type Description
show.bs.collapse show 方法调用之后立即触发该事件。
shown.bs.collapse 此事件在折叠对象显示（并且同时在 CSS 过渡效果完成）之后被触发。
hide.bs.collapse hide 方法调用之后立即触发该事件。
hidden.bs.collapse 此事件在折叠对象隐藏（并且同时在 CSS 过渡效果完成）之后被触发。 
$('#myCollapsible').on('hidden.bs.collapse', function () {
  // do something…
})

用法

多个轮播

多个轮播用id来区分，且要保证id实在每一个轮播的最外层。

通过 data 属性

使用 data 属性可以很容易的控制轮播的位置。data-slide 内可以包含 prev 或 next ，他们可以改变当前帧。另外，使用 data-slide-to 可以传递某个帧的下标，例如 data-slide-to="2" ，这样就可以直接跳转到这个指定的帧 -- 下标从 0 开始计算。

通过 JavaScript

手工启动轮播：

$('.carousel').carousel()

参数设置

参数可以通过 data 属性或 JavaScript 传递。对于 data 属性，将参数名称附着到 data- 之后，例如 data-interval=""。

Name type default description
interval number 5000 轮播切换到下一个的时间，单位是毫秒（ms）。
pause string "hover" 是否启用当鼠标在当前轮播上暂停轮播。
wrap boolean true 轮播是否循环还是停止。

方法

.carousel(options)

初始化一个带有可选参数object的轮播，并且自动开始循环播放。

$('.carousel').carousel({
  interval: 2000
})
.carousel('cycle')

从左向右循环播放。

.carousel('pause')

停止循环播放。

.carousel(number)

循环到指定帧（下标从0开始，类似数组）。

.carousel('prev')

返回到上一帧。

.carousel('next')

跳转到下一帧。

事件

Event Type Description
slide.bs.carousel 当 slide 实例方法被调用时，此事件会被立即触发。
slid.bs.carousel 当切换完一帧后触发。 
$('#myCarousel').on('slide.bs.carousel', function () {
  // do something…
})

返回顶部插件

LXUI集成了返回顶部的插件，只需要使用 .backTop的 <div> 即可。

实例

右下角的返回顶部按钮就是内置的，想要实现此功能首先需要在 <body> 内加上一个 <div> ，如下例：

<!-- 返回顶部的层 -->
<div class="backTop"></div>

<!-- 返回顶部事件 -->
<script type="text/javascript">
  $('.backTop').click(function(){
    $('html,body').animate({scrollTop: '0px'}, 800);
  })
</script>

用法

需要给返回顶部按钮一个点击事件，scrollTop 是返回距离顶部的距离。800 是动画的时间，单位是毫秒(ms)。 

$('.backTop').click(function(){
  $('html,body').animate({scrollTop: '0px'}, 800);
})

无动画效果的返回菜单

只需要加一个 <a> 标签，添加href="#"属性即可。（无动画效果）

<a href="#" class="return-top">返回顶部</a>
